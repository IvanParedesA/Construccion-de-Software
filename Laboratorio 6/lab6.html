Preguntas

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    • Porque si el usuario ha cometido algún error al rellenar el formulario, se le puede notificar de forma instantánea, sin necesidad de esperar la respuesta del servidor


  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    • Por medio de la consola del desarrollador en nuestro navegador, podemos alterar el código e introducir por ejemplo el comando novalidate="" o formnovalidate="" para saltar estas validacciones. Es decir, esto puede hacerse con tan solo introducir un par de palabras reservadas en el código html.


  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    • Porque no cualquier usuario puede saltarse las validaciones así que para la mayoría de la gente una validacion en Javascript será suficiente para que no introduzcan datos erroneos, además de que es muy eficiente hacer la validación del lado del cliente.
