  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • La diferencia es que Internet es el nombre que recibe la conexión que hay entre todas las computadoras alrededor del mundo, mientras que la World Wide Web es la colección de páginas que existe en esa red de computadoras.

  • ¿Cuáles son las partes de un URL?

    • El URL está dividido por diferentes partes, cada una con su función específica. De acuerdo con Edix (2022), las partes de un URL son las siguientes:

    • Esquema: es el protocolo de inicio empleado para la transmisión de datos entre ordenadores y la recuperación de la información de internet. En la mayor parte de los casos el esquema se corresponde con: http o https.
    • Host: normalmente, para el protocolo http, el host es: www.
    • Dominio: corresponde con el nombre del dominio de cada sitio web, como: edix.com.
    • Puerto: define el nombre del puerto según el host. Para el protocolo http, el puerto por defecto es 80, pero esta parte suele omitirse.
    • Directorio (o ruta): define el camino que se debe seguir en el servidor. Por ejemplo: blog.
    • Nombre de archivo: define más detalladamente las características del documento: videos-gratis

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Cuando nos referimos a los métodos, estamos hablando de un formato de comunicación entre el cliente y el servidor web. De acuerdo con Williams Morales (2021), este es el propósito de cada uno de estos métodos:

    • GET: Realiza una petición a un recurso específico y no permite el envío de datos a excepción si dichos datos se envían como parámetro en la Url que realiza la petición. Esta petición retorna tanto la cabecera como el contenido. puede retornar una respuesta en formato HTML, JSON, XML, Imágenes o JavaScript.
    • HEAD: Es parecido al método GET, con la diferencia de que este no retorna ningún contenido HTTP Response. Se utiliza normalmente cuando solo se está interesado en el código de HTTP de Respuesta y el encabezado (Headers). Cuando se tiene un sitio web con varios enlaces web y se requiere verificar si hay “enlaces rotos”; entonces se realiza la petición a todos los enlaces correspondientes y verifica el estado de dichos enlaces.
    • POST: Permite enviar datos al servidor por medio del cuerpo (body) y nada por la Url como se emplea en el método GET. El tipo de cuerpo de solicitud se define en la cabecera Content-Type. Semánticamente se utiliza para registrar información, similar al INSERT de datos a nivel de base de datos. A pesar de eso, se puede forzar este método de petición HTTP para otras acciones como actualización, eliminación de registro, como carga de archivos, etc.
    • PUT: Es similar al método de petición POST, solo que el método PUT es idempotente; es decir puede ser ejecutado varias veces y tiene el mismo efecto. Semánticamente, se utiliza para la actualización de información existente, es semejante a un UPDATE de datos a nivel de base de datos. Los requests de un PUT usualmente se envían los datos por formularios, formato JSON entre otros.
    • PATCH: Este método se emplea para modificaciones parciales de un recurso en particular. Se debe revisar si el servidor es compatible. Para averiguar si el servidor aceptar peticiones PATCH notifica su compatibilidad en el header Allow o Access-Control-Allow-Methods, otra indicación de que estas peticiones están permitidas son el header Accept-Patch. Semánticamente, realiza una especie de UPDATE a nivel de base de datos.
    • DELETE: Este método de petición permite eliminar un recurso específico. También es idempotente; es decir puede ser ejecutado varias veces y tiene el mismo efecto similar al PUT y GET. Semánticamente se utiliza para eliminar de información existente.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Se debe usar el método POST, pues este es el que ofrece más discreción. De acuerdo con IONOS (2020), con este método los datos no se muestran en el caché ni tampoco en el historial de navegación. La flexibilidad del método POST también resulta muy útil: no solo se pueden enviar textos cortos, sino también otros tipos de información, como fotos o vídeos.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • El método que se utiliza es GET.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • No significa que haya ocurrido un error, sino todo lo contrario, el código 200 indica que la solicitud ha sido un éxito.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • No es responsabilidad del desarrollador corregir como tal el sitio web ya que este código de error sale cuando el contenido buscado de la web ha sido eliminado o modificado (intensionalmente), por lo que es probable que lo que se busca esté ahora en una URL distinta. Normalmente puede suceder porque el usuario ha tecleado mal el URL o introdujo uno antiguo sin el conocimiento de que este cambió. Más bien, lo que puede hacer el desarrollador es notificar previamente a los usuarios.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • En este caso sí es responsabilidad del desarrollador corregir este error ya que lo que significa es que ha habido un problema con el servidor del sitio web. Por esta razón, los usuarios de la página no pueden hacer nada para arreglar el error más que esperar a que sea reparado.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Cuando un atributo está desaprobado significa que con la nueva actualización de Html, este ha sido reemplazado por nuevos elementos que hacen lo mismo o mejoran las funciones. Sin embargo, muchos navegadores siguen siendo compatibles con los atributos desaprobados hasta que estos finalmente se vuelven obsoletos. Algunos ejemplos son los siguientes:

    • applet: Reemplazado por el tag HTML object.
    • dir: Reemplazado por el tag HTML ul.
    • isindex: Reemplazado por el tag HTML input.
    • menu: Reemplazado por el tag HTML ul.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • HTML4 tuvo su lanzamiento en 1999 y HTML5 está disponible desde 2014. Con HTML5 hubo pocos cambios para las etiquetas de marcado salvo por algunas que fueron simplificadas. Sin embargo, HTML5 se basa en un estándar y un análisis sintactico nuevos. Entre algunas de las características que lo diferencian es que HTML5 incluye elementos multimedia, es más seguro y esta es una versión de HTML que permanecerá actualizandose a lo largo del tiempo, así que ya no se tiene planeado un HTML6. Además, con las nuevas funcionalidades, HTML5 ya no es completamente dependiente de CSS o JavaScript.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • De acuerdo con (De León, s.f.) los componentes del elemento table son los siguientes:

    • caption: Provee un título para la tabla
    • colgroup: agrupa columnas temáticamente
    • col: aplica atributos comunes y estilos a una o más columnas. Sólo puede ser declarado como hijo de colgroup.
    • tbody: representa a un cuerpo de la tabla.
    • thead: representa la sección de encabezado de la tabla.
    • tfoot: representa la sección de pie de la tabla.
    • tr: representa una fila. Puede ser declarado como hijo de tbody, thead, tfoot o table.
    • td: representa a una celda regular. Sólo puede ser declarado como hijo de tr.
    • th: representa a una celda de encabezado. Sólo puede ser declarado como hijo de tr.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • Los principales controles son los siguientes:

    • number: campo de entrada para la introducción de un número
    • range: permite elegir entre un rango de valores
    • date: selector de fecha
    • color: permite al usuario elegir entre una selección de colores
    • datalist: permite a los usuarios elegir entre una lista de opciones sugeridas
    • tel: campo de entrada para un número telefónico
    • email: campo de entrada para un email
    • url: campo de entrada para un url
    • text: campo de entrada para texto
    • time: campo de entrada para una hora específica

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • Un sisema de información atraviesa 7 fases durante su ciclo de vida, los cuales están enlistados a continuación:

    1. Fase de planificación: se prepara el diseño y posterior implementación del sistema.
    2. Fase de análisis: se busca concretar una serie de requisitos, que son los que regirán el nuevo sistema o los cambios a introducir en el antiguo.
    3. Fase de diseño: se identifican soluciones potenciales, se evalúan y se elige la más conveniente.
    4. Fase de desarrollo: Inicio de la producción.
    5. Fase de integración y periodo de pruebas: corroborar que el diseño propuesto cumple con los requisitos de negocio establecidos.
    6. Fase de implementación: instalación del hardware y software elegidos, crear las aplicaciones correspondientes, someterlas a pruebas, crear la documentación pertinente y capacitar a los usuarios.
    7. Fase de mantenimiento: consiste en introducir los ajustes necesarios para mejorar el rendimiento y corregir los problemas que puedan surgir.

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • El cilo de desarrollo de un sistema de información empieza por analizar los tipos de requerimientos del mismo:

    • Funcionales: requisitos necesarios para hacer que el sistema funcione. Ej: registrar usuarios, loguearse, acceder a un listado de clientes.
    • No funcionales: son los que no están directamente relacionados con la funcionalidad del sistema. Ej: el sistema deberá utilizar navegador Mozilla Firefox versión n o más.

    Los siguientes pasos están directamente relacionados con su desarrollo:

    • Fase de planeación: se especifica un plan para el proyecto que guiará a todos los interesados o stakeholders a conseguir el producto especificado en el levantamiento de requerimientos.
    • Fase de diseño: los analistas o ingenieros de software crean un modelo abstracto del sistema a construir basado en los requerimientos funcionales y no funcionales planteados en la especificación de requerimientos. proporciona detalles sobre arquitectura del software, estructuras de datos, interfaces y componentes que se necesitan para implementar el sistema.
    • Fase de desarrollo: se genera el código fuente en el lenguaje de programación escogido.
    • Fase de implementación: se lleva a cabo la puesta en marcha del producto de software.
